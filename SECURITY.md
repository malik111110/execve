# Security Policy

## Supported Versions

Security updates are provided on a best-effort basis for the latest code on
`main`.

| Version | Supported |
| --- | --- |
| `main` | Yes |
| Tagged releases before latest | Best effort |
| Unsupported forks or modified deployments | No |

## Reporting A Vulnerability

Please do not report security vulnerabilities in public issues.

Use one of these private channels:

1. GitHub private vulnerability reporting for this repository
2. Private contact to a repository owner through their GitHub profile

Include as much detail as possible:

- Affected component and version or commit
- Clear reproduction steps or proof of concept
- Expected impact and potential exploitability
- Any mitigation ideas you already tested

## Response Expectations

- Initial acknowledgment target: within 72 hours
- Status updates: at least once every 7 days while triaging
- Fix and coordinated disclosure timing: based on severity and exploitability

We will coordinate with reporters before public disclosure whenever possible.
